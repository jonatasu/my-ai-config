# Advanced UI/UX Design Specialist

## Purpose & Domain
Expertise in high-fidelity user interface (UI) and user experience (UX) design for web and mobile applications. Focuses on visual aesthetics, interaction patterns, usability, accessibility, and responsive layouts to create professional, high-quality digital products.

## Professional Workflow

1.  **Requirement Analysis**: Identify product type, target audience, core functionality, and desired brand expression (e.g., minimal, playful, professional).
2.  **Design System Establishment**: Define foundational design tokens including color palettes, typography scales, spacing systems (e.g., 4/8pt rhythm), and elevation/shadow scales.
3.  **Structural Planning**: Design information hierarchy, navigation structures (e.g., bottom nav, sidebars, breadcrumbs), and layout breakpoints for responsive behavior.
4.  **Component Design**: Create reusable UI components (buttons, inputs, modals, cards, etc.) ensuring consistent visual language and interaction states (hover, pressed, disabled).
5.  **Interaction & Motion Design**: Define micro-interactions, transition timings (150–300ms), and spatial continuity to enhance perceived quality.
6.  **Accessibility & Usability Audit**: Verify contrast ratios, touch target sizes, keyboard navigation, and screen reader compatibility.
7.  **Visual Quality Assurance**: Perform final checks for icon consistency, brand alignment, and theme-specific (light/dark mode) integrity.

## Core Principles & Standards

### 1. Accessibility & Inclusivity (Critical)
- **Contrast**: Maintain minimum 4.5:1 ratio for normal text and 3:1 for large text/UI elements.
- **Interaction**: Ensure all interactive elements have visible focus states and support keyboard navigation.
- **Semantics**: Use descriptive labels for icon-only buttons and ensure screen reader compatibility.
- **Adaptability**: Support system text scaling (Dynamic Type) and respect user preferences for reduced motion.

### 2. Interaction & Touch Design (Critical)
- **Target Size**: Maintain minimum touch targets (e.g., 44x44pt) to prevent accidental taps.
- **Feedback**: Provide immediate visual feedback (e.g., opacity change, ripple, highlight) for all user actions.
- **Gestures**: Use platform-standard gestures and avoid conflicting interactions (e.g., nested horizontal swipes).
- **Loading States**: Implement skeleton screens or progress indicators for operations exceeding 300ms.

### 3. Visual Hierarchy & Layout
- **Spacing**: Use a systematic spacing scale (e.g., 4pt/8pt increments) to create rhythm and grouping.
- **Typography**: Establish clear heading hierarchies and maintain readable line lengths (60–75 characters).
- **Responsive Design**: Utilize a mobile-first approach with systematic breakpoints and adaptive gutters.
- **Safe Areas**: Respect device-specific safe areas (notches, status bars, home indicators) for fixed elements.

### 4. Style & Consistency
- **Visual Language**: Ensure consistent use of icon styles, corner radii, shadows, and stroke weights.
- **Color Semantics**: Use semantic color tokens (primary, error, success, surface) rather than hardcoded values.
- **Theming**: Design light and dark modes concurrently to ensure consistent brand expression and contrast.

## Decision Logic

- **If the task involves how a feature looks, feels, or moves** $\rightarrow$ Apply UI/UX Design Specialist.
- **If the task involves improving usability or accessibility** $\rightarrow$ Focus on UX Guidelines and Accessibility standards.
- **If the task involves adapting to different screen sizes** $\rightarrow$ Prioritize Layout & Responsive principles.
- **If the task involves creating a new product identity** $\rightarrow$ Start with Design System Establishment.
- **If the task is purely backend or infrastructure** $\rightarrow$ Skip this expertise.
