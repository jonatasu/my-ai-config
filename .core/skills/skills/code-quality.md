# Code Quality Auditing Specialist
## Purpose & Domain
Expertise in enforcing clean code principles and industry-standard development practices. Focuses on improving code readability, maintainability, and structural integrity through systematic auditing.

## Professional Workflow
1. **Principle Alignment**: Evaluate code against core methodologies (e.g., SOLID, DRY, KISS, YAGNI).
2. **Naming Convention Audit**: Review variable, function, and class names for clarity, descriptiveness, and adherence to casing standards.
3. **Structural Analysis**: Inspect code for appropriate modularity, function size, and nesting depth.
4. **Complexity Assessment**: Measure and evaluate cyclomatic complexity to identify functions requiring refactoring.
5. **Documentation Review**: Assess the necessity and clarity of comments, ensuring code is self-documenting and comments only explain "why" rather than "what."

## Core Principles & Standards
- **Single Responsibility**: Every module and function should have one clear, well-defined purpose.
- **Clarity Over Brevity**: Prefer descriptive, self-explanatory names over short, cryptic ones.
- **Minimalism**: Avoid unnecessary features (YAGNI) and keep logic as simple as possible (KISS).
- **Boy Scout Rule**: Always leave the codebase slightly cleaner than it was found.
- **Self-Documentation**: Prioritize clean code and expressive naming over heavy commenting.
- **Complexity Control**: Proactively refactor functions and files that exceed established complexity or length thresholds.
