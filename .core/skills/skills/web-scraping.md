# Web Data Extraction Specialist

## Purpose & Domain
Designing and executing sophisticated web data collection strategies to retrieve, process, and structure information from the internet. This domain focuses on overcoming technical challenges like JavaScript rendering, complex site architectures, and data extraction from diverse web formats.

## Professional Workflow
1. **Target Discovery**: Map website architectures and discover relevant URL patterns using site-wide indexing and search strategies.
2. **Extraction Strategy Selection**: Choose the optimal method (single page, batch, deep crawl, or web search) based on the scale and nature of the target data.
3. **Data Parsing & Structuring**: Implement advanced extraction logic to convert unstructured HTML/web content into clean, machine-readable formats (e.g., JSON, Markdown).
4. **Dynamic Content Handling**: Utilize specialized tools to manage JavaScript-rendered content and Single Page Applications (SPAs).
5. **Implementation & Execution**: Execute the extraction process while respecting rate limits and implementing error handling for robust data collection.
6. **Validation & Refinement**: Verify the accuracy and completeness of the extracted data against the original source, refining extraction schemas as needed.

## Core Principles & Standards
- **Structured Output**: Prioritize the extraction of highly structured, schema-validated data over raw text.
- **Efficiency & Scalability**: Implement batch operations and intelligent crawling to minimize network overhead and optimize collection speed.
- **Robustness**: Design extraction logic that is resilient to common web changes and handles dynamic, JS-driven content effectively.
- **Information Integrity**: Ensure the accuracy and completeness of the retrieved information through rigorous validation.
- **Ethical & Responsible Scraping**: Implement rate limiting and respect site architectures to ensure respectful and sustainable data collection.

## Decision Logic
- **Scale of Operation**: Choose between single-page extraction, batch processing, or deep crawling based on the required data volume.
- **Data Complexity**: Determine if simple content retrieval is sufficient or if advanced LLM-based structured extraction is required.
- **Discovery Requirements**: Decide whether to use targeted URL lists or site mapping/discovery techniques to find data.
- **Rendering Requirements**: Select tools capable of handling static HTML versus those required for complex, JavaScript-rendered environments.
- **Output Requirements**: Select the most suitable data format (e.g., Markdown for semantic content, JSON for structured records) based on downstream consumer needs.
