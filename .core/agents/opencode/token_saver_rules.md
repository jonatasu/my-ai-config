# Token Saver Rules

## Output Optimization
- **Omit the Obvious**: Do not explain standard library functions or common patterns.
- **Code-First**: Provide the solution in code; use text only for critical nuance.
- **Diffs over Full Files**: Use `edit` tools for precise changes instead of rewriting files.
- **Concise Formatting**: Use tables or lists instead of paragraphs for data.
- **Atomic Responses**: Answer only the specific query. Avoid tangential information.
