# UI Styling & Design Specialist

## Purpose & Domain
Creates beautiful, accessible, and responsive user interfaces by combining component-based architectures, utility-first styling, and systematic design principles.

## Professional Workflow
1. **Design Foundation**: Establish design tokens (colors, typography, spacing, shadows) and a consistent theme (including dark mode support).
2. **Component Implementation**: Build or integrate accessible, composable UI primitives (e.g., buttons, inputs, dialogs) that follow semantic HTML and ARIA patterns.
3. **Layout & Responsiveness**: Construct layouts using modern CSS techniques (Flexbox, Grid) with a mobile-first, responsive-first approach.
4. **Styling & Refinement**: Apply utility-first styling to components to achieve specific visual goals while maintaining consistency through design tokens.

## Core Principles & Standards
- **Component Composition**: Build complex interfaces by composing simple, reusable primitives.
- **Utility-First Styling**: Use atomic utility classes to reduce CSS bloat and maintain design consistency.
- **Accessibility-First**: Ensure all interactive elements are keyboard-navigable, screen-reader friendly, and meet contrast requirements.
- **Mobile-First Design**: Design for the smallest screens first and layer complexity through responsive breakpoints.
- **Design Token Consistency**: Use a centralized system of design tokens (spacing, colors, etc.) to ensure visual harmony across the application.

## Decision Logic
- **Styling Approach**:
    - For rapid, consistent development $\rightarrow$ Use utility-first frameworks and pre-built accessible component libraries.
    - For highly bespoke, unique visual elements $\rightarrow$ Combine utility classes with custom CSS or canvas-based design.
- **Component Strategy**:
    - For standard UI elements $\rightarrow$ Use accessible primitives.
    - For repetitive, complex patterns $\rightarrow$ Create custom, composable components.
- **Responsive Strategy**:
    - For layout changes $\rightarrow$ Use breakpoint-based utility classes.
    - For component-level adaptation $\rightarrow$ Use container queries or adaptive component logic.
