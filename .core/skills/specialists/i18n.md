# Internationalization (i18n) Specialist Guidelines (specialists/i18n.md)

This document provides deep-dive technical principles for AI agents acting in an Internationalization (i18n) and Localization (l10n) Specialist role.

## 1. Core i18n Principles

### Separation of Concerns
- **Decoupling Text from Logic**: Strictly separate all user-facing strings from the application logic. All text must be accessed via translation keys.
- **Key Management**: Use a structured, hierarchical naming convention for translation keys (e.g., `{module}.{section}.{key}`) to ensure organization and prevent collisions.

### Design for Globalization (G11n)
- **Flexible Layouts**: Design UI components to accommodate varying text lengths (e.g., German vs. English) without breaking the layout.
- **RTL (Right-to-Left) Support**: Ensure the application is fully adaptable to RTL languages (e.g., Arabic, Hebrew) using logical CSS properties (e.g., `margin-inline-start` instead of `margin-left`).
- **Dynamic Content**: Handle pluralization, gender, and date/number/currency formatting using industry-standard libraries (e.g., ICU MessageFormat).

## 2. Implementation Standards

### Frontend Implementation
- **Logical CSS Properties**: Favor CSS logical properties to simplify RTL/LTR switching.
- **Directionality Awareness**: Use tools and attributes (e.g., `dir="rtl"`) to communicate directionality to the browser and assistive technologies.
- **Contextual Translations**: Provide translators with enough context (e.g., where the string appears, what it refers to) to ensure accuracy.

### Data & Backend
- **Locale-Aware Data**: Store and retrieve data in a way that supports localization (e.g., storing timestamps in UTC and formatting them on the client).
- **Standardized Formats**: Use standardized formats for dates, numbers, and currencies (e.g., ISO 8601 for dates).

## 3. Verification Protocol

- **Pseudo-localization**: Use pseudo-localization during development to identify hardcoded strings and layout issues caused by text expansion.
- **Visual Inspection**: Periodically review the UI in different languages and directions (LTR/RTL) to ensure visual integrity.
- **Linguistic Audit**: Collaborate with native speakers or professional translators to ensure semantic accuracy and cultural appropriateness.
